# Выпускная квалификационная работа (ВКР)

## Содержание документа

Файл **VKR_Kursovaya.md** содержит полный текст ВКР по теме «Разработка официального сайта администрации Сергиево-Посадского городского округа» в формате Markdown:

- **ВВЕДЕНИЕ** — актуальность, объект, предмет, цель, задачи, методы
- **ОСНОВНАЯ ЧАСТЬ** — главы 1–6 (теория, проектирование IDEF0/DFD/ER, реализация, БД, детализация диаграмм, дополнительные аспекты)
- **ЗАКЛЮЧЕНИЕ**
- **Структура БД** — описание таблиц (users, roles, articles, appeals, administration_heads/deputies, vacancies и др.)
- **Диаграммы** — текстовое описание трёх и более диаграмм: IDEF0 (контекстная и декомпозиция, дуги: верх 2, низ 2, право 1, лево 3), DFD (уровни 0 и 1), ER

## Как получить файл .docx

### Вариант 1: Pandoc (рекомендуется)

1. Установите [Pandoc](https://pandoc.org/installing.html).
2. В папке `docs` выполните:
   ```bash
   pandoc VKR_Kursovaya.md -o VKR_Kursovaya.docx --reference-doc=reference.docx
   ```
   Если нет файла `reference.docx` с настройками стилей:
   ```bash
   pandoc VKR_Kursovaya.md -o VKR_Kursovaya.docx
   ```
3. Откройте `VKR_Kursovaya.docx` в Word и при необходимости отформатируйте заголовки, нумерацию страниц, список литературы по ГОСТ.

### Вариант 2: Microsoft Word

1. Откройте Word.
2. Файл → Открыть → выберите `VKR_Kursovaya.md`.
3. Word импортирует Markdown (заголовки станут стилями). Сохраните как документ Word (.docx).

### Вариант 3: Копирование в Word

Откройте `VKR_Kursovaya.md` в любом редакторе и скопируйте содержимое в новый документ Word, затем примените стили «Заголовок 1», «Заголовок 2» и т.д. по уровням (# и ##).

## Доведение до 60+ страниц основной части

Методичка требует не менее 60 страниц основной части. Рекомендуется:

1. **Вставить графические диаграммы** — нарисовать IDEF0, DFD и ER в draw.io, BPwin или Visio по описаниям из текста и вставить рисунки в раздел 2.
2. **Расширить главы** — добавить подпункты в гл. 1 (история сайтов ОМСУ, сравнение технологий), в гл. 3 — пошаговое описание реализации каждого раздела с фрагментами кода.
3. **Добавить разделы** — «Тестирование», «Рекомендации по развёртыванию», «Охрана труда» (если требуется по специальности).
4. **Приложения** — скриншоты главной страницы, формы обращений, кабинета сотрудника; листинги ключевых файлов.
5. **Список источников** — оформить по ГОСТ 7.1 (20–40 наименований): ФЗ, методички, учебники, документация Laravel.
